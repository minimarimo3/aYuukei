# Yuukei 完成版実装仕様書 v1.1
## 03. UIと状態管理

## 9. 画面仕様

設定画面は **単一ウィンドウ + サイドバー切り替え** とする。

### 9.1 サイドバー項目

- 外見と振る舞い
- パッケージ
- マーケットプレイス
- 連携
- 動作設定
- About

### 9.2 外見と振る舞い

対象項目。

- 台本
- VRM
- 小物
- テクスチャ
- ロード済みモーション一覧

各項目には以下の切替を持つ。

- パッケージ準拠
- 個別指定

### 9.3 パッケージ

- 現在有効パッケージ表示
- インストール済みパッケージ一覧
- パッケージ切替
- ローカルインポート
- 削除

### 9.4 マーケットプレイス

- 画面の存在のみ定義
- 実ダウンロード処理は未実装でよい

### 9.5 連携

- 他デバイス連携画面
- LLM 用 API キー入力欄
- 同期状態表示の最低限領域
- 送信有無 / 外部依存の明示領域

### 9.6 動作設定

- 一時無効化 / 一時非表示の説明
- ショートカットキー設定
- 起動時挙動
- 基本演出設定

### 9.7 About

- アプリ名
- バージョン
- ライセンス
- クレジット

### 9.8 UI の MVP デフォルト

- 即時適用でよい
- Apply / Cancel は必須でない
- 未設定項目は「未設定」表示でよい
- DLL 画面には危険性警告と明示確認 UI を置く
- API キー画面には何が外部送信されうるかを表示する
- 選択肢 UI は単純な縦並びボタンでよい

## 12. 無効化・停止状態

### 12.1 一時無効化

- 常駐は続く
- キャラクターは表示されたままでもよい
- 反応 / 台本実行 / 自発動作を停止する
- 状態は保存される

### 12.2 一時非表示

- 常駐は続く
- キャラクター描画を停止する
- 再表示可能
- 集中用途やゲーム用途を想定する

### 12.3 完全終了

- 永続データを保存してプロセス終了する

---

## 19. DLL と安全性

- DLL は危険を伴う拡張として扱う
- 読み込み前に明確な警告 UI を表示する
- 失敗時や危険判定時は自動で読み込まない
- ユーザーの明示確認があるまで保留とする
- API キー未設定時は該当機能のみ無効化
- キャラクター体験自体は継続する
- API キー画面や連携画面には外部送信可能性を表示する
