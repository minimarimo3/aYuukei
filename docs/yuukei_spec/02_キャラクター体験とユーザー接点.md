## 02. キャラクター体験とユーザー接点

## 6. キャラクター挙動仕様

### 6.1 移動

- キャラクターは基本的にデスクトップ上をうろつく / 漂う
- 移動感は「水中をふわふわ漂う」印象を目指す
- 台本や内部状態によって定位置滞在や遊動停止も可能
- マルチモニタは一つの連続空間として扱う
- 障害なくモニタ間移動できること

### 6.2 忙しさ反映

- キー入力やマウス操作が激しいときは移動頻度または移動距離を落とす
- 完全停止でなくてもよい
- フルスクリーン中 / 特定アプリ使用中ディスプレイへ侵入しない
- MVP では内部 busy 状態を持てばよく、公開イベント化は必須でない

### 6.3 発話

- 吹き出しは常に 1 つまで
- 新しい発話が来たら既存吹き出しを上書き
- 発話後、直後に次がなければ速やかに消す
- 音声再生は MVP 対象外
- 口パクは VRM モデル機能の範囲で行う

### 6.4 ファイルとの接触

- MVP で台本に渡すのはファイル名、拡張子、種別のみ
- ファイル内容解析、画像認識、テキスト内容参照は対象外
- D&D は日常操作が接点になることを示す重要入力として扱う

## 7. ユーザー接点

### 7.1 管理 UI への入口

管理 UI への入口は以下のみ。

- 常在アイコン
- ショートカットキー

常在アイコンから可能な操作。

- 設定画面を開く
- 一時無効化
- 一時非表示
- 再表示 / 再有効化
- 完全終了

### 7.2 キャラクター本体への入口

キャラクター本体への入力は台本駆動の出来事として扱う。

- クリック
- ダブルクリック
- ドラッグ開始
- ドラッグ終了
- ファイルドロップ
- 放置
- 定期発火

## 8. 初回起動体験

### 8.1 起動シーケンス

初回起動時は以下の順で進行する。

1. スプラッシュ表示
2. デフォルトキャラクター出現
3. デフォルト Daihon 群有効化
4. キャラクターが物語的チュートリアルを開始

### 8.2 チュートリアル方針

- 専用 UI 主導ではなく Daihon で進行する
- 基本操作をキャラクターとのやりとりの中で学ばせる
- 設定画面の詳細説明は最小限でよい
