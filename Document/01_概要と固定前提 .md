# Yuukei 完成版実装仕様書 v1.1
## 01. 概要と固定前提

## 1. この文書の目的

この文書は、Unity で実装するデスクトップマスコットアプリ **Yuukei** について、追加の対話なしでも Windows 向け MVP を実装できるようにするための完成版実装仕様である。  
本仕様は、プロダクト思想、固定前提、画面仕様、パッケージ仕様、保存仕様、Daihon 実行契約、別名解決、実装単位、失敗時の扱い、MVP の受け入れ条件を定義する。

本仕様が優先するものは以下。

1. ユーザーにとって自然で邪魔しない体験
2. Daihon による振る舞い記述のしやすさ
3. 実装者が迷わないこと
4. 将来拡張を壊さないこと

---

## 2. 固定実装前提

### 2.1 Unity / ランタイム前提

- Unity: **6000.3.6f1**
- Render Pipeline: **URP**
- 非同期処理: **UniTask**
- 透明ウィンドウ / 前面表示制御: **UniWindowController**

### 2.2 対象プラットフォーム

- 将来的な対象: Windows / macOS / Android / iOS / Linux
- **MVP の開発対象は Windows**
- ほかの OS 向け分岐は将来拡張を見越した抽象化だけ残せばよい

### 2.3 Windows MVP で最低限成立させること

- 常駐アプリとして起動できる
- キャラクターを透明ウィンドウ上に表示できる
- マルチモニタ空間を移動できる
- フルスクリーンアプリのディスプレイを避けられる
- 特定アプリ使用中ディスプレイを避けられる
- 常在アイコンから管理 UI と終了にアクセスできる
- グローバルショートカットで一時無効化 / 一時非表示を切り替えられる

### 2.4 OS 連携の最低方針

- ドラッグ中はキャラクター本体の当たり判定を無視してよい
- Alt+Tab 非表示やネイティブ最適化の細部は MVP 必須にしない
- クリック透過は必要最小限で実装する
- 透明表示、前面表示、クリック透過は UniWindowController を優先利用する

---

## 3. プロダクト定義

Yuukei は単なる会話アプリではない。  
**PC の中にキャラクターが住み着いているように感じられる体験を提供する居住型キャラクター体験の実行基盤** である。

重要なのは、AI の高度さそのものではなく、以下を成立させること。

- キャラクターがそこにいると感じられる連続性
- PC 上の自然な文脈に反応すること
- クリック、ドラッグ、ファイル接触、時間経過などが接点になること
- 台本、VRM、モーション、小物、テクスチャ、拡張を差し替えられること

LLM は拡張要素であり、未設定でもアプリ体験は成立しなければならない。

---

## 4. 設計原則

### 4.1 主役は管理 UI ではなくキャラクター体験

- キャラクターをクリックして設定画面を開く設計は禁止する
- キャラクターへのクリックやダブルクリックは、すべて台本上の出来事として扱う
- 設定や終了は常在アイコンやショートカットから行う

### 4.2 本体は実行基盤に徹する

- 振る舞いの大半は Daihon 側に定義する
- アプリ本体はイベント発火、関数実装、表示制御、保存、ロード、安全管理を担当する
- 性格づけや台詞内容は台本責務とする

### 4.3 邪魔しないことを優先する

- 忙しいときは露出と移動を抑える
- フルスクリーンアプリやゲーム中は、そのディスプレイに出ない
- 特定アプリ使用中のディスプレイに侵入しない
- 完全に消すのではなく、様子をうかがうような控えめさを優先する

### 4.4 文脈整合性を優先する

- そのタイミングでその反応をする理由があることを優先する
- 唐突なランダム発話、しつこい反復、文脈に合わない反応は禁止する

### 4.5 拡張しやすさはコア要件

- 台本 / VRM / モーション / テクスチャ / 小物 / DLL は差し替え可能
- パッケージは世界観の一括適用単位
- 非技術者でも台本で振る舞いを育てられる方向を維持する

---

## 5. MVP の到達目標

MVP では以下を成立させる。

- スプラッシュ後にキャラクターが住み着く演出がある
- デフォルトキャラが画面上に出現し、Daihon で振る舞う
- クリック、ダブルクリック、ドラッグ、ファイル D&D、放置、定期発火が動作する
- 常在アイコンから設定画面と終了にアクセスできる
- 設定画面骨格が存在する
- パッケージ切替と個別差し替えができる
- 保存と再起動復元ができる
- 一時無効化 / 一時非表示 / 完全終了が成立する
- マーケットプレイス / 他デバイス連携 / API キー画面は存在する
- DLL 警告付きロード導線が存在する

MVP に含めないもの。

- マーケットプレイス本実装
- 他デバイス同期本実装
- 高度な DLL 管理
- 大規模共有機能
- 高度な自律エージェント挙動
- ファイル内容解析
- 外部音声合成との本統合
